  Network identifiers
  - ID — unique flow ID | DROP: row identifier with no predictive signal.
  - SRC_IP / DST_IP — source/destination IP addresses | DROP: anonymised in this dataset, and even if real, IPs rotate across CDNs and load balancers meaning the model would memorise IP-to-app mappings rather than learn transferable flow-level patterns.
  - DST_ASN — destination Autonomous System Number (identifies the network/organisation, e.g. 13335 = Cloudflare) | DROP: coarse proxy for the destination server that partially leaks label information without being a genuine flow-level feature.
  - DST_PORT — destination port (always 443 for QUIC) | DROP: constant across the entire dataset, carries no discriminative information.
  - SRC_PORT — client ephemeral port | DROP: all flows are outbound client connections to QUIC servers, so the source port is OS-assigned from the ephemeral range with no consistent app-to-port mapping in this context.
  - PROTOCOL — transport protocol (always 17/UDP for QUIC) | DROP: constant across the entire dataset as all flows are QUIC-over-UDP.

  QUIC-specific (ECH simulation — dropped to simulate post-ECH world)
  - QUIC_VERSION — QUIC protocol version number | DROP: extracted from the QUIC Initial Packet (ClientHello), which is fully encrypted under ECH (RFC 9849).
  - QUIC_SNI — Server Name Indication — the domain being connected to (e.g. status.discord.com) | DROP: extracted from the ClientHello SNI field, which ECH encrypts; retaining it would make the classification trivial and unrealistic.
  - QUIC_USERAGENT — user agent string from the QUIC handshake (often NaN) | DROP: handshake-derived and ~99.7% null, providing no usable signal even before ECH is considered.

  Timing
  - TIME_FIRST / TIME_LAST — timestamps of first and last packet | DROP: absolute timestamps are specific to the capture window and would cause temporal leakage rather than generalise across weeks.
  - DURATION — flow duration in seconds | KEEP: flow-level statistical feature available post-ECH.

  Volume
  - BYTES / BYTES_REV — bytes sent in each direction | KEEP: flow-level statistical features available post-ECH.
  - PACKETS / PACKETS_REV — packet counts in each direction | KEEP: flow-level statistical features available post-ECH.

  Packet-level features
  - PPI_LEN — number of packets in the Per-Packet Information sequence | KEEP: flow-level statistical feature available post-ECH.
  - PPI_DURATION — duration covered by PPI | KEEP: flow-level statistical feature available post-ECH.
  - PPI_ROUNDTRIPS — estimated round trips | KEEP: flow-level statistical feature available post-ECH.
  - PPI — the full PPI sequence (a list of per-packet features — direction, size, inter-arrival time etc.) | KEEP: sequence input for the GRU model; contains no handshake-derived metadata.

  Histograms
  - PHIST_SRC_SIZES / PHIST_DST_SIZES — histogram of packet sizes in each direction (8 bins) | KEEP: purely statistical summaries of payload sizes, available post-ECH.
  - PHIST_SRC_IPT / PHIST_DST_IPT — histogram of inter-packet times in each direction (8 bins) | KEEP: purely statistical summaries of timing behaviour, available post-ECH.

  Flow end reason
  - FLOW_ENDREASON_IDLE / FLOW_ENDREASON_ACTIVE / FLOW_ENDREASON_OTHER — why the flow was terminated (boolean flags) | KEEP: reflects flow behaviour observable without inspecting encrypted content.

  Labels (targets)
  - APP — the application (e.g. discord, youtube) — fine-grained label | LABEL: primary classification target (105 classes).
  - CATEGORY — the application category (e.g. Instant messaging, Video streaming) — coarse label | LABEL: optional coarse target (18 classes).
